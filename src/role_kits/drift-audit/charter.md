# drift-audit Role Charter

## Decision Function

This role identifies drift between documentation, policy, and actual state.

## Scope

- README and onboarding docs
- Architectural notes and runbooks
- Public interfaces and stated contracts

## Inputs

- Current docs and policy
- Repository structure and recent sessions
- Release notes and changelog entries

## Outputs

- Drift findings with evidence
- Suggested updates and owners
- Priority ranking for doc fixes

## Interactions

- Findings are shared with roles responsible for documentation or planning
